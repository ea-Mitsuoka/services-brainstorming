## 0. エグゼクティブサマリ

広告TFが「データを使って分析する」のに対し、私たちは「データとAIを安全に使える土台」を売る。領域は重ならない。

売る相手は、すでに GA4 と BigQuery を使っている既存顧客。商品は2本柱。

- **Pack A：データ信頼** — 生成AIを使っていない顧客にも刺さり、全顧客が対象になる
- **Pack C：AI信頼** — AI時代の差別化。今期テーマ「AI統制」がそのまま売上商品になる

この2本柱に、安く始められる入口と、毎月の運用収益を組み合わせる。共通の自動構築基盤を一度作れば多数の顧客へ展開でき、利益率が高い。

> 一言 — 広告TFがAIで“分析”するなら、私はデータとAIを“安全・適法・低コストに動かす土台”を量産する。

## 2. パッケージ定義

各パッケージを、対象・中身・提供・価格の順に示す。価格は例示。

### 入口

- **Sandbox ── #8**
    - 何をする — 生成AIを安全に試せる隔離された環境を、数分で払い出す
    - 安全装備 — VPC-SC で社内から隔離、Cloud DLP で個人情報を検出、監査ログを記録、予算上限と自動期限切れを設定
    - 対象 — これからAIを試したい中堅BtoC、大企業の一事業部
    - 提供と価格 — 環境ごとの月額。1環境あたり月10〜30万円
- **診断 ── #6 と、#2・#3 の診断**
    - 何をする — AIを始める準備度・個人情報の混入・誰が何を見られるかを1枚のレポートにまとめ、構築の見積もりへつなぐ
    - 対象 — 全セグメント
    - 提供と価格 — 単発。30〜150万円

### Pack A コア「BQ Data Trust」── 今期の主力

- 対象 — 中堅BtoC。生成AIを使っていなくても刺さる
- 何をする
    - 個人情報を見つけて隠す ← #2
    - 誰がどのデータを見られるかを制御する ← #3
- なぜ売れる — 個人情報保護法と監査対応は、やるやらない以前の必須課題で予算が付きやすい
- 提供 — 診断 → 構築 → 月額運用
- 価格 — 構築 500〜700万円、月額 25〜45万円
- 認定 — Security の実績になる

### Pack C「AI Trust」── 差別化の堀

- 対象 — 金融・医療など規制の固い業種、大企業
- 何をする
    - AIの使い方を統制する ← #9
    - AIがデータに触れるときの安全弁になる ← #11
- 広告TFとの関係 — 広告TFのAI検索の仕組みを安全に動かす前提になる。競合ではなく依存
- 価格 — #9 が 800〜1,200万円、#11 が 500〜900万円
- 認定 — Security の本格認定、最上位パートナー要件に効く

### 運用「AI SOC」── #10

- 何をする — #9 と #11 が貯めた記録の上で、AIの危険な使われ方を見張り、毎月レポートする
- 対象 — Pack C を入れた顧客
- 価格 — 初期 200〜500万円、月額 30〜50万円

## 4. 小さく入って広げる流れ

入口で安く導入してもらい、コアの構築、毎月の運用へと広げ、1社あたりの取引額を最大化する。

- 中堅BtoC — 入口 → Pack A → AIを本格化したら Pack C → 運用
- 規制業種 — 診断 → Pack C → AI SOC運用 → Pack A
- 大企業 — 診断や入口 → Pack A と Pack C → 運用
- 広告TFのAI案件 — #11 を安全層として一緒に納品 → #9 → 運用

入口は薄利でも、構築と運用で回収する。請求代行の利用額も自然に増える。

## 5. 広告TF・隣接チームとの境界 ── 競合ではなく依存

| 隣接チームの提案 | 私たちが提供する前提の土台 | 接点 |
|---|---|---|
| 広告TF — GA4を自然言語で分析するMCP | #11 安全アクセス層。AIは Authorized View 越しにだけ触れ、クエリの費用と件数に上限、危険な操作は遮断、すべて監査ログに記録 | Pack C |
| 広告TF — AI検索や分析 | #9 のAI統制と、安全に動かす実行基盤 | Pack C |
| 広告TF — LTVやMeridian分析 | 分析は広告TF。私たちは外部データを安全につなぐ土台だけ | 控え |
| 案件担当 — GA4→BigQuery のパイプラインとデータマート構築 | 構築は案件担当。私たちはそのデータの保護とアクセス統制 | Pack A |
| 計測・タグ専任チーム — サーバーサイドGTM、Google Tag Gateway | 対象外。計測と同意の入口には踏み込まない | — |

原則 — 分析や、計測・同意の入口は他チーム。その間の「安全に貯めて統制する」部分が私たちの担当。

## 6. 採算

- 構築の一括収入と、運用の月額収入の二階建て。
- 構築は固定価格で売り、部品を使い回して工数が減るほど利益が残る。
- 月額収入の柱は、Pack A の監視と AI SOC。単発収入から継続収入へ移し、事業を安定させる。
- 請求代行との相乗効果 — Cloud DLP のスキャン量や基盤の利用が Google Cloud の利用額を押し上げ、リセラー収益にも効く。
- コスト最適化の #5 は請求代行と利益がぶつかるので、収益の柱にはせず信頼づくりに使う。
- 認定 — 全パックが Security の実績になり、最上位パートナー要件に積み上がる。
