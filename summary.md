## 0. エグゼクティブサマリ

広告TFが「データを使って分析する」のに対し、私たちは「データとAIを安全に使える土台」を売る。領域は重ならない。
売る相手は、すでに GA4 と BigQuery を使っている既存顧客。商品は2本柱。

- **Pack A ＝ DSPM**（Data Security Posture Management＝データの安全姿勢管理）の **BigQuery/GA4 特化版**。「機微データを 見つける→分類→誰が触れるか→どこへ流れるか」を継続管理する。生成AIを使っていない顧客にも刺さり、全顧客が対象になる予算の付くホット領域。
- **Pack B ＝ AI-SPM / AI-TRiSM**（AIの安全姿勢管理）。新興カテゴリで競合が薄く、先行できる。 AI時代の差別化。今期テーマ「AI統制」がそのまま売上商品になる。

この2本柱に、安く始められる入口と、毎月の運用収益を組み合わせる。共通の自動構築基盤を一度作れば多数の顧客へ展開でき、利益率が高い。

Pack B（AI-SPM）は4つの階層で AI利用を丸ごと固める。

| 階層 | 担当 | 役割（平易に） |
|---|---|---|
| 制御 | B1 | どんなAI/エージェントがあり、何を許すか（棚卸し・採点・ポリシー） |
| データ強制 | B2 | AIがデータに触れる瞬間を安全弁で止める（出力検査・目的制限） |
| **ループ強制** | B3 | **自律エージェントの暴走を時間軸で止める**（緊急停止・予算上限・人手承認） |
| 検知・対応 | B4 | 危険を見つけ、B3/B2と連携して**実際に止める**（Managed AI-DR） |

> 1. B3 自律エージェント（AIが自分で連続作業する仕組み）の運用統制は**まだ市場に空白**で、ここで先行する。
> 1. **CNAPP（Wiz/Prisma/SCC 等のクラウド総合セキュリティ）とは競合せず補完**。CNAPP が手薄な「BQの深いデータ文脈」と「AIランタイム」を、GCP特化・ベンダー中立で埋める。
> 1. 広告TFがAIで“分析”するなら、私はデータとAIを“安全・適法・低コストに動かす土台”を量産する。

## 1. 製品体系（全体像）

![製品体系](products.png)

3つの階層を、共通の自動構築基盤の上に積む。

- **入口** — 安く広く始めてもらう
- **コア2パック** — 価値の本体。Pack A と Pack B
- **運用** — 毎月の継続収益にする

コア2パックは2種類のターゲットを対象とする。

- Pack A は生成AIの利用を問わず、GA4 と BigQuery を使う全顧客が対象
- Pack B はAI時代の差別化で、今期テーマの商品化そのもの

入口〜運用まで全て共通基盤の上に乗り、部品を使い回して横展開するので利益率が高い。

## 2. パッケージ定義

各パッケージを、対象・中身・提供・価格の順に示す。価格は例示。

### 入口

- **E1 Sandbox〔#8〕**
  - 何をする — 生成AIを安全に試せる隔離された環境を、数分で払い出す
  - 安全装備 — VPC-SC で社内から隔離、Cloud DLP で個人情報を検出、監査ログを記録、予算上限と自動期限切れを設定
  - 対象 — これからAIを試したい中堅BtoC、大企業の一事業部
  - 提供と価格 — 環境ごとの月額。1環境あたり**月8〜15万円**（運用の自動化で成立させる）
- **E2 診断〔#6〕（＋A1・A2 の診断）**
  - 何をする — 個人情報の混入・誰が何を見られるかを自動スクリプトで検査し、1枚のレポートに。刺さった顧客は手動分析つきの詳細診断 → 構築見積へ
  - 対象 — 全セグメント
  - 提供と価格 — **自動スクリーニングは無償**（既存顧客限定・工数1〜2人日の自動版のみ）。**詳細診断は30〜50万円**、AI Ready 診断（②③向け）は50〜100万円

### Pack A コア「BQ Data Trust ＝ DSPM for BigQuery」── 今期の主力

- 対象 — 中堅BtoC。生成AIを使っていなくても刺さる
- 何をする
  - 個人情報を見つけて隠す ← A1〔#2〕。DSPMとして、データのリスク採点・流れの地図化・GA4特化の検出まで
  - 誰がどのデータを見られるかを制御する ← A2〔#3〕。誰が実際に触れるかの実効分析・過剰権限の削減＝CIEM for BQ
- なぜ売れる — 個人情報保護法と監査対応は、やるやらない以前の必須課題で予算が付きやすい
- 提供 — 無償診断 → **Step 1「Light」＝A1 のみ → Step 2「A2 追加」** → 月額運用。①の稟議予算帯（1件 100〜300万円）に**各回を収める分割販売**が基本。一括は上位オプション
- 価格 — PII保護 80〜180万円 ／ アクセス統制 150〜220万円 ／ 一括 300〜400万円 ／ 月額 10〜18万円（複数社共有の監視基盤で成立）
- 認定 — Security の実績になる

### Pack B「AI Trust ＝ AI-SPM」── 差別化の堀（4階層）

- 対象 — 金融・医療など規制の固い業種、大企業
- 何をする（4つの階層で固める）
  - AIの使い方を統制する（棚卸し・採点・ポリシー） ← B1〔#9〕
  - AIがデータに触れる瞬間の安全弁（出力検査・目的制限） ← B2〔#11〕
  - **自律エージェントの暴走を止める**（緊急停止・予算上限・人手承認） ← B3〔#14〕（新）
  - 危険を検知して B3/B2 で実際に止める ← B4〔#10〕（運用、下記）
- 広告TFとの関係 — 広告TFのAI検索の仕組みを安全に動かす前提になる。競合ではなく依存
- 価格 — **2段構え**：新カテゴリで実績がまだ無いため、初期は参入価格（B1 500〜800万円／B2 350〜600万円／B3 構築200〜400万円＋月額20〜40万円）で実績を取り、リファレンス獲得後に段階的に引き上げる
- 認定 — Security の本格認定、最上位パートナー要件に効く

### 運用「Managed AI-DR（AI検知・対応）」── B4〔#10〕

- 何をする — B1・B2・B3 が貯めた記録の上で、AIの危険な使われ方（乗っ取り・情報持ち出し・暴走）を見張り、**B3/B2 と連携して実際に止める**。脅威の分類は業界標準（OWASP Top10 for LLM／MITRE ATLAS）に沿う。マネージド（当方運用）で毎月レポート
- 対象 — Pack B を入れた顧客
- 価格 — 初期 150〜300万円、月額 25〜40万円（24時間・自動対応は上位）

## 4. 小さく入って広げる流れ

入口で安く導入してもらい、コアの構築、毎月の運用へと広げ、1社あたりの取引額を最大化する。

- 中堅BtoC — 入口 → Pack A → AIを本格化したら Pack B → 運用
- 規制業種 — 診断 → Pack B → Managed AI-DR運用 → Pack A
- 大企業 — 診断や入口 → Pack A と Pack B → 運用
- 広告TFのAI案件 — B2 を安全層として一緒に納品 → B1 → 運用

入口は薄利でも、構築と運用で回収する。請求代行の利用額も自然に増える。

## 5. 広告TF・隣接チームとの境界 ── 競合ではなく依存

| 隣接チームの提案 | 私たちが提供する前提の土台 | 接点 |
|---|---|---|
| 広告TF — GA4を自然言語で分析するMCP | B2 安全アクセス層。AIは Authorized View 越しにだけ触れ、クエリの費用と件数に上限、危険な操作は遮断、すべて監査ログに記録 | Pack B |
| 広告TF — AI検索や分析 | B1 のAI統制と、安全に動かす実行基盤 | Pack B |
| 広告TF — LTVやMeridian分析 | 分析は広告TF。私たちは外部データを安全につなぐ土台だけ | 控え |
| 案件担当 — GA4→BigQuery のパイプラインとデータマート構築 | 構築は案件担当。私たちはそのデータの保護とアクセス統制 | Pack A |
| 計測・タグ専任チーム — サーバーサイドGTM、Google Tag Gateway | 対象外。計測と同意の入口には踏み込まない | — |

原則 — 分析や、計測・同意の入口は他チーム。その間の「安全に貯めて統制する」部分が私たちの担当。

## 6. 採算

- 構築の一括収入と、運用の月額収入の二階建て。
- 構築は固定価格で売り、部品を使い回して工数が減るほど利益が残る。
- **価格改定（D18）の帰結** — ①の買える価格に下げた結果、今期は投資年（▲650万円規模）。来期に横展開の自動化・月額の積み上げ・Pack B の高単価で黒字化する。「今期＝商品化投資、来期＝回収」のストーリーで説明する。
- 月額収入の柱は、Pack A の監視と Managed AI-DR（B4）。単発収入から継続収入へ移し、事業を安定させる。
- 請求代行との相乗効果 — Cloud DLP のスキャン量や基盤の利用が Google Cloud の利用額を押し上げ、リセラー収益にも効く。
- コスト最適化の A4〔#5〕 は請求代行と利益がぶつかるので、収益の柱にはせず信頼づくりに使う。
- 認定 — 全パックが Security の実績になり、最上位パートナー要件に積み上がる。
