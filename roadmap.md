### 技術スタック（GCP）

- **データ／統制**：BigQuery、Cloud DLP、Dataplex、Data Catalog、Authorized View、Row/Column-level Security
- **基盤／境界**：IAM、Org Policy、VPC Service Controls、KMS、Secret Manager、Cloud Asset Inventory、Security Command Center
- **可観測性**：Cloud Logging から BQ Sink、Cloud Monitoring、Looker / Looker Studio
- **CI/CD**：GitHub Actions or Cloud Build、Workload Identity Federation、tfsec / Checkov / Conftest、Terraform。tfstate は GCS バックエンドで状態ロック
- **AI**：Vertex AI、Gemini。Pack B で MCP/Agent ガードレール

### CI/CDパイプライン（全パック共通）

`PR → fmt/validate → tfsec/Checkov → Conftest で Policy as Code → terraform plan を PRコメント → 承認 → WIF経由 apply → 監査ログ/ドリフト検知`

> **設計原則**
>
> 1. 全部品はモジュール化し、顧客別は変数だけで差し替える
> 1. 機密は Secret Manager ＋ WIF の鍵レス運用
> 1. Org Policy・VPC-SC はコード化し、逸脱を CI で止める
> 1. Looker監査テンプレは全パック共通で再利用する

## 単価前提とユニットエコノミクス ── 人日 ¥10万 基準

> 標準単価 ＝ ¥100,000/人日（8時間）。本書では 1人月 ＝ 20人日 ＝ ¥200万 で換算する。
> この単価は工数の金額換算＝原価・見積の最低ライン。販売価格はこの上に乗せる固定価格とする。

### 価格設計の原則 ── 構築は「人日精算」ではなく「固定価格」で売る

- 純粋な人日精算（工数 × ¥10万）にすると、モジュール再利用で工数が減るほど売上も減り、横展開の旨味が顧客側に流れてしまう
- そこで構築パック（Pack A/B、控えの Landing Zone 含む）はバリュー固定価格で販売する。工数削減分は自社の粗利に変わる。これが「一度作って横展開で粗利が跳ねる」を成立させる
- 診断・スポット改修・運用 MRR は人日精算でも固定でも可
