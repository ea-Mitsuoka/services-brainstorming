# Secure Data & AI Foundation — 事業ロードマップ

> 推奨構造「**1ブランド × 3パッケージ × 3フェーズ**」を時系列に落としたもの。
> 着手は **Pack A（PII保護）** から。共通IaCエンジンを育てながら B→C へ拡張する。
> ※ 月は 2026/06 起点の目安。決算期（今期末）に合わせて調整可能（下記はApr–Mar決算を仮定）。

## 全体像

```
 1つの旗（ブランド）: Secure Data & AI Foundation
   └ 共通エンジン: Terraformモジュール群 + CI/CD + Looker監査テンプレ（1回作って横展開）
        ├ Pack A: Data Trust（PII保護・ガバナンス・品質・コスト） … 最も広い入口・生成AI不問
        ├ Pack B: Foundation / Landing Zone（IaC一括構築）       … 本命SI
        └ Pack C: AI Trust（AIガバナンス・SOC・安全アクセス層）   … 差別化の堀・広告TF MCPの前提
   売り方（横串の3フェーズ）: 診断 Assess → 構築 Build → 運用 Operate(MRR)
```

## フェーズ別ロードマップ

### Phase 0 — 旗を立てる / PoC素地（〜2026/8・約1〜2ヶ月）

- **動き**：ブランド定義、広告TFとの棲み分けを上司・社内で合意。社内GA4/BQデータで **PII検出PoC**（URL・クエリパラメータへのメール混入を実証＝刺さるデモ化）。共通IaCリポジトリ＋CI/CD骨格を着手。
- **製品**：— （仕込み）
- **収益**：0（投資フェーズ）
- **スペシャライゼーション**：Security向け実績の設計開始
- **次へのゲート**：PoCデモ完成 ＋ 社内棲み分け合意

### Phase 1 — Pack A 最小商品化 → 初受注（2026/8〜11・今期前半）

- **動き**：#2 PII保護を中核に Terraformモジュール化（DLP/Dataplex/マスキング/Authorized View）。軽量アセスメント「データ安全性診断」を入口商品として整備。既存GAリセラー顧客 数社へ診断提供 → 構築受注。
- **製品**：**Pack A（Data Trust）** 提供開始 ／ 診断メニュー
- **収益**：診断フィー ＋ 最初の構築SI
- **スペシャライゼーション**：Security 実績の積み上げ開始
- **次へのゲート**：有償構築 1〜2社受注 ＋ 再利用可能モジュール化完了

### Phase 2 — Pack B（Landing Zone）+ 運用MRR 立ち上げ（2026/11〜2027/3・今期後半〜今期末）

- **動き**：#7 Landing Zone を A の延長で構築（GA4→BQ→Looker→Vertex AI の安全基盤）。#1品質監視・#3アクセスガバナンスをモジュール追加。\*\*月額運用（Managed Service / MRR）\*\*提供開始。共通エンジン再利用で2社目以降の構築工数を圧縮。
- **製品**：**Pack B（Foundation/Landing Zone）** ＋ **運用サービス（MRR）**
- **収益**：構築SI（主力）＋ 月額ストック収益の発生
- **スペシャライゼーション**：Data Analytics / Infrastructure の申請実績に到達を狙う
- **次へのゲート**：MRR顧客 数社 ＋ 横展開で構築工数 30%以上削減を実証

### Phase 3 — Pack C（AI Trust）で差別化・堀を作る（2027/4〜9・来期前半）

- **動き**：#9 AI Governance Platform（今期テーマの商品化）。#11 AI安全アクセス層 を **広告TFのMCPサーバーの前提条件**として社内連携・クロスセル。#10 AI SOC を運用メニュー追加。広告TF案件とのセット提案（相互送客）が回り始める。
- **製品**：**Pack C（AI Trust）**
- **収益**：高単価SI ＋ 運用MRRの上積み
- **スペシャライゼーション**：Security 本格認定／プレミアパートナー要件への寄与
- **次へのゲート**：広告TF案件との共同提案 成立 ＋ AIガバナンス受注

### Phase 4 — スケール / プロダクト化（2027/10〜）

- **動き**：3パック横断の標準プロダクト化、リファレンス事例化、既存数百社基盤への横展開を加速。
- **製品**：3パック統合プロダクト
- **収益**：SI＋MRRの両輪でスケール
- **スペシャライゼーション**：複数領域の認定維持・拡大

## 収益モデルの推移（イメージ）

- Phase 1：**診断 → 構築SI**（フロー収益中心）
- Phase 2：構築SI ＋ **MRR発生**（ストック化開始）
- Phase 3〜4：SI ＋ **MRR積み上げ**（ストック比率上昇＝事業の安定化）

## 前提・リスク

- 決算期により月割りは要調整（上記はApr–Mar仮定）。
- 広告TFとの棲み分け合意（特に Pack C / Clean Room）が政治的な前提。
- 営業・実装リソース確保。Pack A から軽く始め、実績で投資を正当化する設計。
- 応用層（予測#12・自動レポート#13）は広告TF重複のため本ロードマップに含めない。

詳細な13アイデアの整理は `ideas_new_services.md` を参照。
